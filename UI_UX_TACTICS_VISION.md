# UI/UX Vision: Тактична секція cs-utility
> Бачення для `/team` — одноручне управління, максимальна швидкість, мінімум когнітивного навантаження у матчі.

**Оновлення (травень 2026):** після першого MVP зафіксовано ревізію продукту — див. `PLAN_TEAM_TACTICS.md` §0.1, §5.0, §19. Коротко: **спочатку група і ролі, потім карта/тактика**; обов'язкова **тактична карта**; **миттєвий** перехід до раскидок обраної карти.

---

## 0. Головний принцип: «Телефон на столі або на підставці — рука на мишці»


- **Усі головні дії — в зоні великого пальця** (нижня третина екрану).
- **Нічого критичного у верхньому лівому куті** — туди великий палець не дотягується без перехвату.
- **Мінімум тапів до інформації**: від входу в `/team/[code]` до плану своєї ролі — не більше 2 тапів.
- **Читабельність з відстані 40 см під кутом** — шрифт ≥ 16px для тексту кроків, ≥ 20px для ключових дій.
- **Ніякого пінч-зуму, ніяких горизонтальних свайпів** — тільки вертикальний скрол.

## 1. Дизайн-система: розширення поточного стилю

Поточний застосунок: темний (#0d0d0d фон), CS2-естетика. Тактика її успадковує і розширює.

### 1.1. Кольори ролей

Кожна роль отримує власний акцентний колір — він проходить крізь увесь UI: іконка, пілюля, бордер картки, підсвічування активного кроку.

| Роль | Колір | Логіка |
|---|---|---|
| `entry` | `#FF4444` (червоний) | Перший на прорив — агресія |
| `awp` | `#FFD700` (золотий) | Снайпер — цінний, виокремлений |
| `lurker` | `#9B59B6` (фіолетовий) | Тіньова гра, непередбачуваність |
| `support` | `#27AE60` (зелений) | Підтримка, утиліта |
| `igl` | `#3498DB` (синій) | Командування, лідерство |
| `anchor_a` | `#E67E22` (помаранч.) | Утримання, стабільність |
| `anchor_b` | `#E67E22` + штриховий бордер | Той самий anchor, але B-сайт |

### 1.2. Типографіка кроків

- **Час кроку** (0:30): моноширний шрифт, колір `#666`, розмір 13px — не відволікає.
- **Текст кроку**: 16px, `font-weight: 500`, колір `#E8E8E8`.
- **Callout-пілюля** (назва позиції): 12px, uppercase, колір ролі з 20% opacity фон.
- **Кнопка відео раскидки**: 15px, `font-weight: 700`, повна ширина, 56px висота.

### 1.3. Іконки кроків (`StepKind`)

Прості емодзі → не. Краще SVG-іконки у стилі CS2 HUD:

| Kind | Іконка | Опис |
|---|---|---|
| `spawn` | 🛒 → мінімалістична іконка корзини | Закупка |
| `move` | → стрілка маршруту | Рух |
| `hold` | ⬛ / прицільний квадрат | Тримати кут |
| `throw` | Іконка гранати | Кидок |
| `peek` | Очі / прицільна мітка | Виглядаємо |
| `exec` | ⚡ / вибух | Синхронний вихід |
| `rotate` | 🔄 стрілка | Ротація |
| `note` | Лист / ! | Важлива заметка |

## 2. Флоу капітана — спочатку група, потім карта

### 2.1. Що змінилось після MVP

У першій версії капітан обирав карту/сторону/тактику **до** збору команди. Це суперечить реальному матчу: спочатку п'ятірка в Discord, потім IGL каже «граємо Mirage, дефолт A».

**Новий порядок:**

1. **Створити зустріч** — тільки нік → QR/посилання.
2. **Лобі** — друзі заходять, бачать вільні ролі.
3. **Ролі** — кожен обирає свою (IGL у капітана вже зайнята).
4. **Брифінг** (капітан) — карта → сторона → тактика з бібліотеки **цієї** карти.
5. **План** — текст + карта + раскидки.

### 2.2. «Quick Create» — тільки нік

```
┌──────────────────────────────────────┐
│  ▬  Нова зустріч                     │
├──────────────────────────────────────┤
│  Твій нік: [____________]            │
│                                      │
│  ╔══════════════════════════════╗    │
│  ║    ⚡ Створити зустріч        ║    │  ← 64px
│  ╚══════════════════════════════╝    │
└──────────────────────────────────────┘
```

Без карти, без сторони, без тактики. Капітан одразу потрапляє в лобі з QR.

### 2.3. Брифінг — після ролей (капітан)

Коли ≥2 ролі зайняті, у капітана з'являється PRIMARY **«Обрати карту і тактику»**. Той самий UI пілюль/карток, що раніше був у create-sheet:

```
┌──────────────────────────────────────┐
│  Усі ролі розібрані?                 │
│  Карта  [Mirage] [Dust2] …           │
│  Сторона  [ T ]  [ CT ]              │
│  Тактика  ╔ T Default A ╗            │
│  ╔══════════════════════════════╗    │
│  ║   Зафіксувати для команди    ║    │
│  ╚══════════════════════════════╝    │
└──────────────────────────────────────┘
```

Інші гравці бачать skeleton: «Капітан обирає карту…» → потім «Mirage · T · Default A».

### 2.4. Екран «Готово» (після створення)

Не окрема сторінка — **той самий Bottom Sheet**, що трансформується:

```
┌──────────────────────────────────────┐
│  ✓ Зустріч створена                  │
├──────────────────────────────────────┤
│         ┌───────────┐                │
│         │  QR-код   │                │  ← 180×180px
│         └───────────┘                │
│                                      │
│  Код:  A B 1 2                       │  ← великий моноширний, 40px
│                                      │
│  ╔══════════════════════════════╗    │
│  ║   📋 Скопіювати посилання    ║    │  ← 56px
│  ╚══════════════════════════════╝    │
│  ╔══════════════════════════════╗    │
│  ║   → Перейти в лобі           ║    │  ← secondary, 56px
│  ╚══════════════════════════════╝    │
└──────────────────────────────────────┘
```

Кнопка «Скопіювати» — одноразова дія. Після тапу: ✓ анімація, тост «Посилання скопійоване».

## 3. Флоу гравця — вхід по посиланню

### 3.1. Deep link → одразу до ролі

Гравець тапає по посиланню в Discord. Очікуваний шлях: **відкрилось → вибрав роль → бачу план**.

Нічого зайвого між посиланням і планом. Якщо нік ще не збережений — перший і єдиний блок перед вибором ролі:

```
┌──────────────────────────────────────┐
│  Зустріч AB12                        │
│  ─────────────────────────           │
│  Як тебе кликати?                    │
│  [___________________]  → Далі       │
└──────────────────────────────────────┘
```

Карта й тактика **ще невідомі** — це нормально. Після ніку → вибір ролі → очікування брифінгу → план.

Одне поле + одна кнопка. Ніякої реєстрації.

### 3.2. RolePicker — вибір ролі

**Сітка 2×3 займає весь екран.** Кожна картка — 50% ширини, ~120px висоти:

```
┌────────────┬────────────┐
│  🔴 Entry  │  🟡 AWP    │
│  Перший    │  Снайпер   │
│  входить   │            │
├────────────┼────────────┤
│  🟣 Lurker │  🟢Support │
│  Лурк      │  Утиліта   │
│  через B   │            │
├────────────┼────────────┤
│  🔵 IGL    │            │
│  Капітан   │  (вільно)  │
│  [Ти тут]  │            │
└────────────┴────────────┘
```

- Зайнята роль: картка стає напівпрозорою (40% opacity), поверх — "● Shamil".
- Вибір — моментальний тап без підтвердження. Без діалогів.
- Тактильний відгук (Haptic) при тапі.

**Важливо:** роль `igl` вже зайнята капітаном — на ній одразу видно його нік. Якщо ти капітан — своя картка підсвічена і заблокована від повторного вибору.

## 4. RolePlanView — головний екран матчу

Це найважливіший екран. Гравець дивиться на нього між раундами.

**Два режими перегляду** (вкладки в хедері, один тап):

| Вкладка | Зміст |
|---|---|
| **Текст** | Timeline кроків (як зараз) |
| **Карта** | Радар зі стрілками руху, маркерами гранат, фільтром ролей |

Гравець має бачити **глобальну картину** (куди йде вся команда), а не лише свій список.

```
┌──────────────────────────────────────┐  ← STICKY HEADER
│ Mirage · T  │ 🟣 Lurker  │  [≡]     │  44px
│ [ Текст ]  [ Карта ]                 │  ← перемикач

│  │ 🟣 0:00  🛒 Закупка           │    │

│                                      │
├──────────────────────────────────────┤
│  [✓ Готовий]   [▶ Раунд]  [↻ Знову] │  ← STICKY BOTTOM BAR, 72px
└──────────────────────────────────────┘
```

### 4.2. Sticky Bottom Bar — зона великого пальця

Три кнопки внизу завжди доступні:

| Кнопка | Висота | Дія |
|---|---|---|
| `✓ Готовий` | 56px | Відмітка "я готовий" (для лобі-стану) |
| `▶ Раунд` | 56px | Запускає таймер раунду (1:55 відраховує кроки) |
| `↻ Знову` | 56px | Скидає всі відмітки виконаних кроків |

Кнопка `▶ Раунд` — найширша (50% ширини), бо це головна дія.

### 4.3. Таймер раунду — автопідсвічування кроків

Коли натиснуто `▶ Раунд`, клієнт починає відлік від 1:55:
- Поточний крок (за полем `time`) підсвічується акцентним кольором ролі.
- Ліворуч від нього — маленький анімований пульс `●`.
- Попередні кроки — приглушені (40% opacity), ніби "пройдені".
- **Автоматичний скрол** до активного кроку — не потрібно шукати очима.

### 4.4. Взаємодія з кроком

| Дія | Результат |
|---|---|
| Тап по кроку з `grenade_id` | Відкривається BottomSheet з відео (існуючий компонент) |
| Long-press по будь-якому кроку | Крок відмічається як виконаний (закреслення, opacity 50%) |
| Тап по callout-пілюлі | Відкривається оверлей-картка з поясненням позиції |
| Тап по іконці `kind` | Нічого (декоративно, без поведінки) |

### 4.5. Sticky «🎬 Відео» при виборі кроку

Якщо гравець тапнув на крок, але ще не відкрив відео — внизу над bottom bar з'являється sticky-плашка:

```
┌──────────────────────────────────────┐
│  🎬 Smoke на Окно  →  Відкрити відео │  ← 52px, з'являється з анімацією
└──────────────────────────────────────┘
```

Це дозволяє спочатку прочитати текст кроку, потім окремою дією глянути відео.

### 4.6. Вкладка «Карта» (`TacticMapView`)

```
┌──────────────────────────────────────┐
│ [ Текст ]  [ Карта ]                 │
├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │
│  │         RADAR Mirage           │  │
│  │    ═══► entry (червоний)       │  │
│  │    - - ► lurker (фіолет)       │  │
│  │    💨 smoke @ Window          │  │
│  │    ★ exec target A site        │  │  ← загальна ціль
│  └────────────────────────────────┘  │
│  [ Усі ] [Entry] [Lurker] [Моя]      │  ← фільтр шару
└──────────────────────────────────────┘
```

| Режим | Поведінка |
|---|---|
| **Усі** | Усі 5 маршрутів напівпрозоро + легенда ролей; видно синхронний exec |
| **Роль X** | Тап по чипу — лише маршрут і гранати цієї ролі |
| **Моя** | Дефолт для гравця; акцентний колір ролі |

- Стрілки: `path[]` у кроках `move` / `rotate`.
- Маркери гранат: кроки `throw` + `grenade_id` → іконка типу гранати на радарі.
- Тап маркера → той самий `BottomSheet` з відео раскидки.

## 6. Навігація між відео раскидки і планом

Поточний BottomSheet перевикористовується. Єдина зміна — додати у хедер BottomSheet контекстну breadcrumb-плашку:

```
← Назад до плану  ·  Кидок 0:30  ·  Smoke на Окно
```

Повернення: тап по `← Назад` або свайп вниз по BottomSheet (стандартна поведінка).

URL-стан: `?from=meet:AB12&role=lurker&step=step_3` — дозволяє deep-link і коректний `router.back()`.

## 9. Анти-паттерни (конкретні відмови)

| Що відкидаємо | Чому |
|---|---|
| Модальний діалог «Ви впевнені, що хочете взяти роль AWP?» | Зайвий тап. Взяв — взяв. Є кнопка «Змінити роль». |
| Список кроків як таблиця з колонками | Важко читати однією рукою. Тільки вертикальні картки. |
| Анімації переходів > 200ms | Під час матчу кожна секунда рахується. |
| Спливаючі popover при кожній дії | Перекривають контент. Тільки toast внизу. |
| Горизонтальний свайп між кроками | Губиш місце в плані, незрозуміло "де я". Тільки скрол. |
| Текст ролі як dropdown/select | Потребує двох тапів. Тільки явні картки. |
| Хедер вищий 48px | Краде простір для кроків. |
| Будь-що за межами safe area (notch/home indicator) | Технічний обов'язок, не обговорюється. |

## 10. Прогресивне розкриття (Progressive Disclosure)

Принцип: показуємо мінімум, розкриваємо за потреби.

| Завжди видно | Після тапу/розгортання |
|---|---|
| Текст кроку + час | Деталі callout позиції |
| Іконка kind | — (декоративно) |
| Пілюля callout | Оверлей з поясненням позиції |
| Пілюля «🎬» | BottomSheet з відео |
| Ім'я тактики | Bottom Sheet зі списком тактик |

## 12. Миттєвий міст «Тактика ↔ Раскидки»

### 12.1. Проблема

Повний перехід на `/map/de_mirage` відчувається як «вийшов з тактики» — губиться контекст раунду, довго вантажиться.

### 12.2. Рішення: Session Dock

Постійна смуга внизу (на `/team/[code]` і `/map/[mapId]` при `?meet=AB12`):

```
┌──────────────────────────────────────┐
│   ⚡ Тактика      🎯 Раскидки         │
└──────────────────────────────────────┘
```

- **Prefetch** раскидок одразу після вибору карти на брифінгу.
- **Не розмонтовувати** план при перемиканні (прихована панель або parallel route).
- На сторінці карти заголовок: **«← Тактика»**, не «До списку карт».
- Ціль: перемикання **&lt; 100 ms**, без білого спалаху.

### 12.3. Query-контракт

`?meet=AB12&fromMeet=/team/AB12?plan=1&side=T` — карта знає, куди повертатися.

## 13. Відкриті питання від мене до тебе

1. **Чи потрібна кнопка «Покинути зустріч»** — чи достатньо просто закрити вкладку?
2. **«Раунд розпочато» у лобі** — капітан запускає для всіх, чи кожен для себе?
3. **Тактика без плану для моєї ролі** — показати загальний план (IGL-рівень) чи окремий порожній стан з поясненням?
4. **Ввід ніку**: зберігати в localStorage назавжди, чи питати щоразу для нової зустрічі (параноя щодо спільних пристроїв)?
5. **Що відбувається якщо всього 2-3 гравці** — блокуємо вільні ролі або залишаємо "без гравця"?
6. **Брифінг:** капітан може змінити карту після фіксації, чи потрібен голос команди?
7. **Карта «Усі»:** за замовчуванням показувати всі маршрути чи спочатку «Моя роль»?
8. **Dock раскидок:** окрема вкладка в `/team` чи завжди повноцінний `/map/[id]` під dock?
