## Ответ на вопрос про «гифки вместо роликов»

**Короткий ответ: то что мы запланировали — это и есть «гифки», только в правильной упаковке.**

В современном вебе настоящий `.gif` использовать нельзя:

| Метрика | GIF | MP4 (то, что у нас) |
| --- | --- | --- |
| Размер 5-сек клипа 800×450 | **18.4 МБ** | **680 КБ** (×27 меньше) |
| Скорость декодирования | базовая | **в 7 раз быстрее** |
| Цвета | 256 (много шума на видео) | true color |
| Звук | нет | можно (мы выключаем `-an`) |
| Loop без задержки | да | да (с `loop` атрибутом) |
| Контролы | принудительно нет | можно скрыть |
| Поддержка iOS Safari | да | да (с `playsinline`) |
| Core Web Vitals / SEO | плохо | отлично |

В коде BottomSheet **уже стоит** `<video autoplay loop muted playsinline>`. С точки зрения UX это **визуально неотличимо от GIF**: автоплей, loop, никаких контролов по умолчанию, без звука. Просто файл в десятки раз меньше и грузится мгновенно.

> Источник цифр: web.dev/articles/replace-gifs-with-videos, css-tricks.com/gifs-without-the-gif, cliptics.com/blog/gif-vs-mp4-vs-webp-2026.

**Поэтому решение зафиксировано:** хранить MP4 (H.264 720p, без звука, 5–15 сек, ~1–3 МБ) — см. `content/INVENTORY.md` §2.1 и `content/lineups/README.md` §2.

## Бонусные находки

### 1. Active Duty pool актуален

С января 2026 (Premier Season 4) пул карт: **Ancient, Anubis, Dust 2, Inferno, Mirage, Nuke, Overpass**. У нас в `src/data/maps.json` ровно эти 7 карт ✅. **Никаких изменений в каталоге карт не требуется.** Vertigo и Train ушли в резерв.

### 2. Готовые радары можно подтянуть из GitHub

Если когда-нибудь потребуется обновить радары — **MurkyYT/cs2-map-icons** делает это автоматически ежедневно. URL прямой:

```
https://raw.githubusercontent.com/MurkyYT/cs2-map-icons/main/images/radars/de_mirage_radar_psd.png
```

То же для thumbnails и иконок CS2.

### 3. Иконки гранат — атрибуция обязательна

Game-icons.net под CC BY 3.0. **Атрибуция требуется**, шаблон `ATTRIBUTION.txt` лежит в `content/assets/README.md`. При экспорте в `public/icons/grenades/` обязательно скопировать его рядом.

### 4. OG-превью можно не хранить как файлы

Next.js 16 умеет `next/og` нативно — генерируем превью на лету через серверный компонент. Не надо рисовать 7 PNG руками. Шаблон — в `content/assets/README.md` §5.2.
