# План: главная в виде карусели (4 вкладки) + поиск по точкам

## Цель

Заменить один блок «Командная тактика» на **четыре равноправных раздела** главного экрана. Переключение: **свайпы влево/вправо** и **тап по кнопке сегмента**. Сохраняем привычку: **точка входа по умолчанию — выбор карты** (текущая сетка карт).

---

## Порядок экранов (слева → направо)

Индексы для разработки (`0 … 3`), визуально карусель горизонтальная:

| Индекс | Раздел        | Содержание |
|--------|---------------|------------|
| 0      | **Поиск**     | Полный экран поиска по точкам / коллаутам → результаты карточками по картам → переход на шаг «Откуда бросаем» |
| 1      | **Выбор карты** | Текущая главная: сетка карт, счётчики раскидок (как сейчас без CTA «Командная тактика») |
| 2      | **Тактики**   | Заглушка → позже командная тактика (см. `PLAN_TEAM_TACTICS.md`) |
| 3      | **Новинки**   | Заглушка → позже вертикальная лента коротких видео / свежих раскидок |

**Дефолт при открытии приложения:** показываем **индекс 1 — Выбор карты**.

**Жест «с главной в поиск»:** с экрана выбора карты **свайп пальцем слева направо** (контент уезжает вправо, справа заезжает левая панель) — открывается **Поиск** (индекс 0). Обратно: **свайп справа налево** с поиска на карту.

> Уточнение для единообразия с ОС: в доке зафиксировать одну схему (какая ось = «назад к карте») и проверить на iOS/Android в прототипе.

---

## Навигация: UI сегментов (4 кнопки)

- Под шапкой `SiteHeader` — **горизонтальный сегмент-контроль** на четыре пункта: иконка + короткая подпись (или только иконки на узком экране с `aria-label`).
- Предлагаемые метки (черновик, i18n позже):
  - Поиск · Карты · Тактики · Новинки
- **Активный** сегмент подсвечивается (золотой акцент, как табы на карте позиций).
- Тап по сегменту: **плавный скролл** карусели к нужной странице (snap).
- Сегменты **синхронизированы** с положением карусели при свайпе (индикатор движется вместе с жестом).

Доступность: `role="tablist"` / `role="tab"` или честный `aria-roledescription="carousel"` + кнопки «вперёд/назад» при желании позже.

## Экран 0 — Поиск

### Назначение

Быстро найти **точку (позицию)** по названию/коллауту (например, «Стэйрс», «Stairs») и перейти к стандартному сценарию **«Откуда бросаем»** на выбранной карте.

### Вход на вкладку

1. Пользователь попадает на экран поиска (свайп или тап по «Поиск»).
2. Верх страницы — **одно поле поиска** на всю ширину (стили как в списке позиций: скругление, тёмный фон).
3. **Фокус в поле сразу** после появления экрана → **открывается клавиатура** (важно для мобилы; на десктопе — курсор в инпуте).

### Поведение при вводе

- Пока запрос **пустой** — под полем **пустое состояние**: короткий текст-подсказка («Введите название точки, например Стэйрс») без фиктивных результатов.
- При вводе текста — **debounce** (например, 200–300 ms) → запрос к данным (клиентский фильтр по загруженным позициям или API, решение на этапе реализации).
- Результаты — **список карточек**. Каждая карточка = **карта + совпавшая позиция + сторона (T или CT)**. Одна и та же точка по названию на одной карте может дать **две карточки** (T и CT), если для обеих сторон есть релевантные раскидки — пользователь явно видит, с какой стороны заходит.

### Дубликаты имён на разных картах

На многих картах есть «Stairs» / «Стэйрс». Каждая карточка в выдаче явно подписана:

- **Крупно:** название точки (локализованный лейбл).
- **Вторично:** **название карты** (например, `Mirage`, `Inferno`).
- **Сторона:** компактная метка **T** или **CT** (или «Террористы» / «Спецназ» в локали) — обязательно, потому что **в одну и ту же точку с T и с CT заходят с разных мест и раскидки разные**.
  - **T** — **оранжево-золотой** акцент, в коде как у террористов в приложении (`#F0B429`, светлый фон `…22`, обводка при необходимости).
  - **CT** — **синий** (`#3B82F6`, тот же приём с фоном/обводкой).
  Визуально — pill рядом с названием карты или под строкой «Mirage», чтобы глаз цеплялся за цвет до тапа.
- Опционально: маленький превью радара / иконка карты.
- Бейдж: число доступных раскидок **на этой карте, для этой точки и этой стороны** (если есть в данных).

Если на одной карте несколько позиций совпали с одним запросом — либо **несколько карточек** с уточнением подзаголовка (категория зоны: «Точка А», «Мид»), либо одна карточка с чипами «2 варианта» — зафиксировать в MVP один простой вариант (предпочтительно **одна строка на тройку map + position + side**).

### Действие «выбор карточки»

- Тап по карточке → **навигация на существующий URL** выбранной позиции с **уже выставленным `side=t` или `side=ct`** из карточки, т.е. шаг **«Откуда бросаем»** (`zone`, `pos` и т.д. — как после выбора точки из списка зоны).
- Отдельный выбор «дефолтной стороны» для всего поиска не нужен: сторона задаётся **каждой карточкой**; при желании позже можно добавить фильтр «Только T / Только CT / Все» над выдачей.

### Верхняя панель

- Кнопка **«Назад»** не обязательна, если сегменты и свайп на карту достаточны; опционально — свернуть клавиатуру по скроллу списка результатов.

## Экран 1 — Выбор карты

- Текущая реализация главной: сетка карт, без большого CTA «Командная тактика» (контент тактики переносится на вкладку 2 позже).
- Вертикальный скролл только внутри этой страницы карусели; **горизонтальный** скролл не конфликтует, если карусель перехватывает жест по оси X с приоритетом.

---

## Экран 2 — Тактики

- MVP: **заглушка** в стиле приложения (иконка, текст «Скоро», ссылка на будущий документ).
- Контент из `PLAN_TEAM_TACTICS.md` подключается на следующем этапе.

---

## Экран 3 — Новинки

- MVP: **заглушка** («Лента новинок скоро»).
- Дальше: вертикальные короткие видео, метаданные (карта, зона, точка, откуда бросок, дата) — отдельный техдизайн.

---

## Технические заметки (без реализации в этом документе)

- **Карусель:** нативный `scroll-snap` + горизонтальный контейнер или библиотека; важно `touch-action` и отсутствие конфликта с вертикальным скроллом на экране карт.
- **Поиск:** переиспользовать логику `positionMatchesSearch` / нормализацию строки как в `PositionList` / `PositionPhotoGrid`.
- **i18n:** все новые строки — ключи в словарях `ru` / `en` / `uk`.
- **Глубина URL (опционально):** `/?tab=search` для шаринга и восстановления состояния.

---

## Критерии готовности MVP

1. Четыре сегмента, свайп + тап, старт с «Выбор карта».
2. Поиск: фокус + клавиатура, выдача карточек с **картой, точкой и стороной (T оранжево-золотой, CT синий)**, переход на «Откуда бросаем» с корректным `side`.
3. Тактики и Новинки — непустые заглушки без битых ссылок.

## Обучалка (onboarding) — **последняя задача эпика**

После того как **карусель главной**, **поиск по точкам**, вкладки **«Тактики»** и **«Новинки»** стабилизированы и отданы пользователям, отдельным **финальным этапом** делаем **обучалку**: короткие подсказки про свайпы между четырьмя разделами, про смысл каждой вкладки, про поиск (пример запроса), про **плашки T / CT** в выдаче. Предусмотреть повторный показ (например из настроек или «Показать снова»). Формат (coach marks, модалка, отдельный слайдер) и тексты — отдельным проходом.

**До этого этапа** отсутствие обучалки **не блокирует** выпуск карусели и поиска: первый опыт строим на понятных подписях сегментов и карточках, обучалку добавляем когда сценарий уже проверен в бою.
