# Гайд для модератора

Это пошаговый гайд для тех, кто наполняет приложение раскидками, фотографиями и видео. UI админки повторяет UX игрока — поэтому если ты понимаешь, как пользователь выбирает раскидку, ты уже понимаешь, как её залить.

> Главный URL: `/admin`. Точки и оверрайды живут в `src/data/custom-lineups.json` и `src/data/position-overrides.json`. Фото и видео — в `public/uploads/grenades/`.

> Журнал изменений админки: **[`ADMIN_CHANGELOG.md`](../ADMIN_CHANGELOG.md)** (в приложении: `/admin/changelog`).

> **Покрытие позиций (обновление):**
>
> - **Mirage** — см. `src/data/positions.ts`: полный набор callout'ов (~22 родителя) + под-точки T/CT спавнов.
> - **Dust2 / Inferno / Nuke / Overpass / Anubis / Ancient / Cache** — спавны (`spawnGroup` в `positions.ts`) + **полная сетка callouts** в `src/data/positions-callouts.ts` (~17–29 зон на карту; общепринятые имена Total CS / CS Console / сообщество). Координаты hotspot на радаре можно уточнять по мере наполнения. У **Nuke** у точек указан `hotspot.layer_id`: `upper` или `lower` под два слоя радара — фильтрация по слою в UI при желании добавляется позже.
>
> Для остальных callout'ов (apartments на Inferno, banana и т.д.) — добавляй позиции вручную в `src/data/positions.ts` по образцу Mirage. На UI это никак не ломает: пока позиции нет, игрок просто её не видит, а карта остаётся доступной через `?all=1` в админке.

## 1. Один взгляд на архитектуру

Каждая раскидка тегируется тремя осями, по которым её ищет игрок:

| Поле | Что значит | Пример |
|------|------------|--------|
| `side` | За кого играешь | `T`, `CT`, `both` |
| `position_ids[]` | Откуда бросают (позиция-callout, может быть несколько) | `t_spawn`, `apartments` |
| `subspot_ids[]` (часть `position_ids[]`) | Конкретная физическая точка появления игрока | `t_spawn_apps_exit` |

Игрок в приложении выбирает: **сторону** → **позицию** → (если есть под-точки) **под-точку** → видит карточки раскидок и тапает нужную.

Чтобы раскидка попала к игроку, она должна иметь те же теги, что игрок выбрал. Админка **проставляет теги автоматически**: ты просто идёшь по тому же flow и тыкаешь на радар.

## 2. Поток модератора (универсальный)

Всегда одна и та же воронка:

```
/admin
  → Mirage              (выбор карты)
    → T                 (выбор стороны)
      → T Spawn         (выбор позиции-callout)
        → Apps Exit     (выбор под-точки — если есть)
          → радар + +Добавить точку → клик / клик
```

На каждом шаге сверху висят pill-чипсы (`T · Терры`, `T Spawn`, `Apps Exit`) с крестиком — нажми крестик, чтобы вернуться к выбору. Кнопка `←` сверху работает по той же логике.

Альтернативный режим — `Все точки` (`/admin/[mapId]?all=1`): без фильтров, ты видишь всё, что есть на карте, и сам проставляешь теги в форме точки. Используй для миграций или массовых правок.

## 3. Шаг A. Карта → Сторона → Позиция

1. Открой `/admin` и тапни плитку карты, например `Mirage`.
2. На экране **Выбери сторону** — `T · Терры` или `CT · Спецы`. Каждая сторона рендерится теми же цветами, что у игрока (T — жёлтый, CT — синий).
3. На экране **Выбери позицию** — список позиций для этой стороны (`T Spawn`, `Apartments`, `Top Mid` и т.д.). Снизу есть пункт `Любая позиция` — для перехвата (точка не попадёт в фильтр позиций, только в фильтр стороны).
4. Тапнул позицию → в URL появилось `?side=t&pos=t_spawn`.

Зачем это нужно тебе: после этого при создании любой точки ты получишь **авто-тег** `side=T` и `position_ids: ['t_spawn']`. Не надо вручную проставлять.

## 4. Шаг B. Под-точка (sub-spot)

Под-точки — это **конкретные физические места появления игрока** внутри одной позиции. Например, на `T Spawn` Mirage их 7: `Apps Exit`, `Palace Side`, `Ramp Side`, `Back Spawn`, `Default`, `Box`, `Corner`.

После выбора позиции, **если у неё есть под-точки**, админка покажет страницу `Откуда стоит игрок?` с фото-карточками 16:9:

- **Тап по карточке** — выбираешь эту под-точку, попадаешь в радар-режим. Все точки, которые ты нарисуешь дальше, получат тег `subspot_ids: ['t_spawn_apps_exit']`.
- **Кнопка `📷 Залить фото` под карточкой** — открывается file picker, ты выбираешь in-game скриншот, он улетает в `/uploads/grenades/`, а в `position-overrides.json` прописывается `screenshot_url`. Фото сразу появляется и у пользователей, и у тебя в интерфейсе. См. [Шаг 7](#7-загрузка-фото-для-под-точки-visual-picker).
- **Любая под-точка** (большая dashed-кнопка сверху) — пропускаешь шаг. Раскидка попадёт только в фильтр родительской позиции, а не конкретной точки. Полезно, если раскидка работает с любого места спавна.

Если у позиции **нет под-точек** — этот шаг скипается, ты сразу попадаешь в радар-режим (как было раньше).

## 5. Шаг C. Накидка (lineup) на радаре

Это сердце админки — экран с радаром и формой редактирования. URL уже содержит весь контекст (`?side=t&pos=t_spawn&spot=t_spawn_apps_exit`), поэтому **новые точки получают теги автоматически**.

### Как добавить раскидку

1. Нажми `+ Добавить точку` (большая жёлтая кнопка справа).
2. Все остальные маркеры на радаре исчезнут, чтобы ты случайно не тапнул чужой. Появится подсказка `👁️ Чужие маркеры скрыты (N)`.
3. Включён режим **двух кликов** (галочка ниже):
   - Шаг 1 — кликни на радаре, **куда падает граната** (target / приём).
   - Шаг 2 — кликни, **откуда бросать** (start point).
4. Точка появилась → справа открылась форма редактирования.

### Форма редактирования

| Поле | Описание |
|------|----------|
| **Название** | Короткое, по-русски: «Smoke Окно», «Flash на Палас». |
| **Тип** | smoke / flash / molotov / HE — иконка маркера на радаре. |
| **Сторона** | Авто, можно поправить. Зелёная подпись `Подставлено из контекста` подсветит, что админка уже за тебя проставила. |
| **Позиции (callouts)** | Мульти-чек. Уже отмечена позиция и под-точка. Можно добавить ещё (если раскидка работает с нескольких мест). Кнопка `Показать все` снимает фильтр стороны. |
| **Общее описание** | Что это за раскидка, для какой ситуации (Default, Mid execute, Retake). |
| **Несколько бросков в одну точку приёма** (чекбокс) | Включи, если из разных мест в одно — будут варианты броска (см. ниже). |

### Один вариант броска vs. несколько

- **Одна точка** (чекбокс выкл.) — у раскидки одна стартовая позиция, одно видео, одна галерея фото.
- **Несколько вариантов броска** — у раскидки **одна точка приёма** (target) и **несколько стартов** (`Бросок 1`, `Бросок 2`, …). У каждого варианта свои:
  - Подпись (`Run boost`, `Stand`, `Crouch jump`).
  - Старт X / Y (или кнопка `Указать старт кликом по радару`).
  - Тип броска: Обычный / С бега / С прыжка / Прыжок + бег.
  - Описание варианта.
  - Своё видео и галерея фото.

Кнопка `+ Ещё вариант броска` копирует последний вариант — обычно начинают копией и правят.

### Горячие действия в форме

- **Указать приём кликом по радару** — после нажатия следующий клик на карте поставит target в кликнутую точку.
- **Указать старт кликом по радару** (внутри варианта) — то же для start.
- **Удалить точку** — стирает всю раскидку.

После всех правок жми **`Сохранить на сервер`** — данные улетают в `src/data/custom-lineups.json`. Игроки увидят их сразу.

## 6. Загрузка фото и видео

В форме раскидки и каждого варианта броска есть блок медиа (компонент `MediaFields`):

- **Видео** — drag&drop или клик по полю → откроется file picker. Допускается `.mp4`, `.webm`, `.mov`. До 120 МБ. После загрузки в поле `video_url` появляется путь `/uploads/grenades/<uuid>.mp4`.
- **Галерея** — несколько фото подряд, разделены переводами строки в textarea. Можно вставить URL с любого CDN или загрузить через тот же file picker. До 25 МБ на фото.

> Технические требования к роликам — [§9](#9-технические-требования-к-контенту).

## 7. Загрузка фото для под-точки (visual picker)

Это отдельная фича — фотография самой **точки появления игрока** (его in-game POV), которая отображается:

- В правой панели у игрока (`SubspotPicker`).
- В вкладке `Фото` главного picker-а позиций (`PositionPhotoGrid`).

### Как залить

1. Идёшь в админку до шага B (`/admin/de_mirage?side=t&pos=t_spawn`).
2. Под нужной карточкой жмёшь `📷 Залить фото` (или `🔁 Заменить фото`, если уже есть).
3. Выбираешь файл (jpeg/png/webp), он:
   - Загружается в `/uploads/grenades/<uuid>.jpg`.
   - В `src/data/position-overrides.json` для `id=t_spawn_apps_exit` прописывается `{"screenshot_url": "/uploads/grenades/<uuid>.jpg"}`.
4. Карточка обновляется автоматически. Игроки сразу увидят фото.

### Что снимать

- В CS2 в **тренировке** или на **локальном сервере**.
- Игрок стоит в точке, **смотрит в направлении площадки** (туда, куда полетит граната с этой точки).
- Без HUD: `cl_drawhud 0` в консоли (потом `cl_drawhud 1`).
- Один кадр (PrintScreen или Steam screenshot — `F12`).

### Параметры файла

| Параметр | Значение |
|----------|----------|
| Соотношение | 16:9 (640×360 или 1280×720) |
| Формат | WebP (приоритет) или PNG |
| Размер | ≤ 200 KB (WebP), ≤ 500 KB (PNG) |
| Имя | произвольное, на сервере получит UUID |

Если у тебя нет WebP — залей PNG, потом сожмём в фоне.

### Удалить фото обратно

Пока — вручную в `src/data/position-overrides.json` (стереть запись для нужного `id`). Кнопка «Удалить фото» — в плане на следующую итерацию.

## 8. Дополнительные сценарии

### Раскидка работает с нескольких под-точек

В форме раскидки в поле **Позиции (callouts)** отметь нужные id чек-боксами. Например, smoke на окно может работать одинаково с `t_spawn_default` и `t_spawn_box` — отмечаешь оба. Раскидка появится в галерее обоих под-точек.

### Раскидка не привязана к под-точке (`Любая под-точка`)

На шаге B нажми `Любая под-точка` (dashed-кнопка). Раскидка попадёт только в фильтр родителя (`t_spawn`), но не в фильтр конкретной spot'ы. Это нормально для базовых defaults, которые работают «откуда угодно с T-спавна».

### Перенести seed-раскидку в свою

Если на радаре есть **базовая** точка (из `grenades.json`, обводка пунктиром), которую ты хочешь дополнить или переписать:

1. Тапни её на радаре → форма справа покажет «База (grenades.json)».
2. Жми `Сохранить как мою точку` — она клонируется в `custom-lineups.json` (с авто-тегами текущего контекста), а оригинал прячется. Дальше редактируешь как обычную свою точку.

### Удалить базовую точку с радара

В той же форме `База` есть кнопка `Удалить` — она просто скрывает базовую точку у тебя локально (фактически: id попадает в `hidden_seed_ids` JSON). Файл `grenades.json` не изменяется.

### Все точки разом, без фильтров

`/admin/[mapId]?all=1` — открывает админку без авто-фильтров. Удобно для редактирования старых точек, у которых ещё не проставлены `position_ids`.

## 9. Технические требования к контенту

### Видео раскидок (`.mp4`)

| Параметр | Значение |
|----------|----------|
| Кодек | H.264 (libx264) |
| Разрешение | 720p (1280×720) — оптимум; 480p тоже ок |
| Длительность | 5–15 секунд |
| Размер | 1–3 МБ; до 5 МБ — приемлемо |
| Звук | **Нет** (вырезаем — не нужен) |
| FPS | 30 |
| Контейнер | MP4 (faststart, чтобы воспроизведение начиналось мгновенно) |

Команда `ffmpeg`:

```bash
ffmpeg -i input.mov \
  -vf "scale=1280:-2" -r 30 \
  -c:v libx264 -profile:v main -crf 24 -preset slow -movflags +faststart \
  -an \
  output.mp4
```

> Подробнее про пайплайн записи — `content/lineups/README.md`.

### Фотографии галереи раскидки

| Параметр | Значение |
|----------|----------|
| Соотношение | 16:9 (для preview-карточки в `LineupGallery`) |
| Формат | WebP (приоритет) или JPEG |
| Размер | ≤ 300 KB |
| Содержание | Точка приёма (target), кросс-хейр на месте отскока, бросок «руками» |

### Фото под-точки (visual picker)

См. [§7](#7-загрузка-фото-для-под-точки-visual-picker).

### In-game скриншоты позиций (главный picker)

То же, что для под-точек, только снимаешь callout (например, в `Apartments` смотришь на жёлтое окно). Загружать пока вручную в `position-overrides.json` или через раздел админки на родительских позициях (этот UI ещё не сделан, в плане на следующий тур).

## 10. Чек-лист QA перед публикацией

Перед тем как сказать «готово» по карте, пройдись:

- [ ] Все позиции для нужной стороны имеют хотя бы одну раскидку.
- [ ] У позиций с под-точками — у каждой под-точки есть фото (`screenshot_url`).
- [ ] Каждая раскидка имеет:
  - [ ] корректное название на русском,
  - [ ] правильный тип (smoke / flash / molotov / HE),
  - [ ] видео < 5 МБ, без звука, начинается мгновенно,
  - [ ] хотя бы одно фото в галерее (для preview-карточки в `LineupGallery`),
  - [ ] описание (что это, в какой ситуации использовать),
  - [ ] корректные `position_ids` (родитель + под-точка, если есть).
- [ ] Тыкаешь раскидку на радаре, открывается BottomSheet с видео — оно играет с первой попытки (особенно на телефоне).
- [ ] Меняешь под-точку через pill в шапке — фильтр на радаре и галерее обновляется.
- [ ] Кнопка ← возвращает шаг назад, а не сразу на главную.

## 11. FAQ

**Где хранится моё содержимое?**

| Тип | Файл / папка |
|-----|--------------|
| Раскидки модератора | `src/data/custom-lineups.json` |
| Скрытые seed-раскидки | `hidden_seed_ids` в том же JSON |
| Фото и видео | `public/uploads/grenades/` |
| Фото под-точек | `src/data/position-overrides.json` (`screenshot_url` под нужным `id`) |
| Карты, радары | `src/data/maps.json`, `public/minimaps/` |
| Каталог позиций | `src/data/positions.ts` |

**Что попадает в Git?**

Всё, кроме сырых видео-захватов. См. `.gitignore` (`content/lineups/_raw/...`). Файлы из `public/uploads/grenades/` коммитим — они нужны в проде.

**Я хочу сделать новую под-точку (например, добавить 8-ю на T-спавне Mirage).**

Сейчас под-точки описаны в `src/data/positions.ts`. Открой, найди родителя (`t_spawn`), скопируй существующий subspot, поправь `id`, `label`, `label_i18n`, `point` (координаты на радаре в [0..1]). UI под-точки в админке появится автоматически. Залей фото через кнопку, как обычно.

В будущем планируется UI для создания/редактирования под-точек прямо в админке — тогда `positions.ts` станет «seed»-файлом, а правки уйдут в JSON.

**Почему две точки у меня на одном радаре, а не одна?**

Скорее всего, у тебя одна **раскидка с несколькими вариантами броска** — она показывается как один маркер `target` + N маркеров `start`. Это правильно: у игрока одна карточка, внутри переключатель `Бросок 1 / 2 / ...`.

**Я залил видео, но ничего не появилось.**

Проверь:

1. Файл реально появился в `public/uploads/grenades/`?
2. В форме раскидки в поле `video_url` стоит путь? (`/uploads/grenades/<uuid>.mp4`)
3. Нажал `Сохранить на сервер`?
4. На вкладке игрока (`/map/[mapId]?...`) обнови страницу. Если видео всё равно не открывается — `404` ли на путь? (Сетевая консоль браузера.)

**Можно ли добавить локализацию названия раскидки?**

Сейчас название хранится одной строкой. Локализация — поверх (например, `name_i18n`) — следующая итерация. Используй русский по умолчанию.

**Под-точки на других картах?**

Сейчас под-точки заведены только для `t_spawn` Mirage (7 placeholder-точек). Для других позиций / карт `subspots` не задано — админка просто пропускает шаг B и сразу даёт радар. Чтобы добавить — расширь `src/data/positions.ts`.

## Полезные пути

| Что | URL / путь |
|-----|------------|
| Главная админки | `/admin` |
| Админка карты (поток) | `/admin/[mapId]` |
| Админка карты без фильтров | `/admin/[mapId]?all=1` |
| Пользовательская страница карты | `/map/[mapId]?side=t&pos=...&spot=...` |
| API раскидок | `POST /api/admin/lineups` |
| API оверрайдов позиций | `POST /api/admin/position-overrides` |
| Загрузка файла | `POST /api/admin/upload` |
| План UI | `PLAN_SIDE_SELECTION_UI.md` |
| Контент-инвентарь | `content/INVENTORY.md` |
| Чек-лист контента | `CONTENT_AND_MEDIA_CHECKLIST.md` |
